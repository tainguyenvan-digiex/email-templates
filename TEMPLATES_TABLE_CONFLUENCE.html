Email Templates Reference Table

This document lists all email templates with their aliases, mappings to the database, and dynamic fields that need to be provided from the server.

Oxen Layout Configuration

All email templates use oxen-layout as the shared layout. This layout features a dark theme (#0c0d14) with silk wave header decoration, DM Sans + Bellfair fonts, and a standardized footer structure.

Location:

  • layouts/oxen-layout.html - Layout HTML template
  • templates/config/oxen-layout/en.json - English footer/config
  • templates/config/oxen-layout/fr.json - French footer/config

Oxen Layout Fields (shared across all templates):

  • website_url - Website URL for logo link
  • help_url - Help centre URL
  • privacy_url - Privacy policy URL
  • terms_url - Terms of service URL
  • footer_automated - Automated message text
  • footer_legal - Legal disclaimer text
  • footer_help - Help link label
  • footer_privacy - Privacy link label
  • footer_terms - Terms link label
  • footer_copyright - Copyright text

Note: Each template provides its own title and subject in its config. Template-specific content fields vary per template. The first_name and recipient_email fields are null in configs and must be provided dynamically by the server.

CTA URLs: Templates with call-to-action buttons require a cta_url field to be provided dynamically by the server. This field should contain the full URL where the button should redirect users.

Common Dynamic Fields

Most templates require these common fields:

  • first_name - User's first name
  • recipient_email - User's email address
  • inviter_name - Name of the person inviting (used in invite-director-access, invite-member-access)
  • company_name - Company/business name (used in invite-director-access, invite-member-access, approve-kyb)
  • otp_code - One-time password code for email verification (used in verify-email)
  • card_last_four - Last 4 digits of card number (used in card-blocked, card-deleted, card-unblocked, high-spending)

Statement/Document Templates

Templates that generate downloadable documents (statements, invoices, RIB) share these fields:

  • customer_name - Customer/account holder name
  • download_url - URL to download the document
  • expiration_time - Link expiration duration
  • expiration_hours - Link expiration in hours

Date/Time Fields

Call-related templates use different date/time field names:

  • Call Confirmed: appointment_date, appointment_time, location
  • Call Reminder: appointment_date, appointment_time, call_link
  • Call Rescheduled: new_date, new_time, location
